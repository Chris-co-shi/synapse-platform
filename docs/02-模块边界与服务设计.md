# Synapse Platform 模块边界与服务拆分设计

版本：TASK-002 Java 微服务骨架版

## 1. 项目边界

`synapse-platform` 是独立平台微服务项目，通过 Maven 依赖复用 `synapse-framework`。

禁止：

- 修改 `synapse-framework`；
- 让 Framework 反向依赖 Platform；
- 在 Platform 中重复实现 Framework 已有技术基础设施；
- 创建 `synapse-platform-bom`、`synapse-platform-common` 或统一 `synapse-platform-api`。

## 3. API / Client / Server 边界

`*-api` 只承载平台微服务内部调用契约。本次只创建 POM 和源码目录，不创建 DTO、事件、错误码或接口。

`*-client` 只承载面向业务系统或第三方系统的 SDK。本次只创建 POM 和源码目录，不创建 Feign Client、自动配置或降级实现。

`*-server` 是可独立启动、注册、配置、部署和扩容的 Java 微服务。本次只创建启动类、配置文件和 Flyway 目录占位。

## 4. 依赖方向

允许：

```text
client -> 对应 api
server -> 对应 api
platform service -> synapse-framework
```

禁止：

```text
api -> client
api -> server
client -> server
server -> 自己的 client
server -> 其他服务的 server
跨服务共享 Entity / Mapper / Repository
跨服务直接访问数据库
循环依赖
```

## 5. Gateway 边界

`synapse-gateway-platform` 是统一入口服务，不拆 api / client / server。

Gateway 使用：

- Spring Cloud Gateway；
- Spring WebFlux；
- Reactor；
- Nacos Discovery；
- Nacos Config；
- Actuator；
- `synapse-core`；
- `synapse-webflux`；
- `synapse-cloud`；
- `synapse-time`。

Gateway 禁止依赖：

- `synapse-webmvc`；
- `synapse-data`；
- `spring-boot-starter-web`；
- `spring-webmvc`；
- Servlet API；
- 任何服务的 Entity / Mapper / server 模块。

默认路由约定：

| --- | --- | --- |
| `/iam/**` | `lb://synapse-iam-server` | `StripPrefix=1` |

## 7. 包结构

Server 根包：

```text
com.indigo.synapse.{module}
```

启动类包：

```text
com.indigo.synapse.{module}.bootstrap
```

本次不生成空 Controller、空 Service、空 Repository 或无意义接口。

## 8. 配置与环境

每个可启动服务包含：

```text
application.yml
application-dev.yml
application-beta.yml
application-prd.yml
```

Nacos Config 按 Spring Boot 3 / Spring Cloud ConfigData 规范通过 `spring.config.import` 加载，不创建 `bootstrap.yml`。

`dev / beta / prd` 隔离 Nacos namespace 和 group。配置中不得提交真实凭证、Token、Secret、私钥或个人路径。

## 9. 验证标准

```bash
mvn validate
mvn clean test
mvn -pl synapse-gateway-platform dependency:tree
```

检查项：

- reactor 识别 50 个项目；
- 13 个启动类存在；
- 52 个 `application*.yml` 存在；
- Gateway 不依赖 `synapse-webmvc` 和 `synapse-data`；
- 不存在 Platform BOM / Common / 统一 API；
- 不存在业务 Controller、Entity、Mapper、Repository、Service 和 Migration。
